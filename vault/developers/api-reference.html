API Reference

Complete reference for the Jutis browser extension provider API. Based on the standard Ethereum provider interface, extended with Canton-specific features.

On this page

info

Static documentation — no live SDK

This documents the provider interface directly. A TypeScript SDK will be released in a future update. Until then, use the provider API as documented below — patterns are stable.

Provider Object

The Jutis extension injects a jutis object into window on every page it serves. This is the primary entry point for all wallet interaction.

TypeScript definition

interface JutisProvider { requestAccounts(): Promise<string[]> getSigner(): JutisSigner getNetwork(): Promise<NetworkInfo> on(event: string, listener: Function): void removeListener(event: string, listener: Function): void disconnect(): Promise<void> } declare global { interface Window { jutis: JutisProvider; } }

Wallet Connection

requestAccounts

Prompts the user to approve your app's connection request. Returns an array of Canton addresses owned by the user.

// Signature requestAccounts(): Promise<string[]> // Example const accounts = await window.jutis.requestAccounts(); // → ["canton1abcd1234..."]
ResultDescription
string[]Array of connected Canton addresses
throws ACTION_REJECTEDUser declined the connection request
throws CHAIN_NOT_FOUNDCanton network not configured in extension

Accounts

getSigner

Returns a JutisSigner instance bound to the currently connected account. All signing operations use this signer.

// Signature getSigner(): JutisSigner // Example const signer = window.jutis.getSigner();

JutisSigner interface

interface JutisSigner { getAddress(): Promise<string> getChainId(): Promise<string> getNetwork(): Promise<string> signMessage(message: string): Promise<string> signTypedData(data: object): Promise<string> sendTransaction(tx: TransactionRequest): Promise<TransactionResponse> }

getAddress

Returns the currently connected Canton address as a string.

const address = await signer.getAddress(); // → "canton1xyz..."

Signing

signMessage

Requests the user to sign a plaintext message. The message is shown as-is in the Jutis confirmation popup.

// Signature signMessage(message: string): Promise<string> // Example const sig = await signer.signMessage("Authenticate with My dApp"); // → "JutisSigned:abc123..."

Returns a prefixed signature string. Verification utility functions will be provided in the SDK.

Network

getNetwork

Returns information about the currently connected Canton network.

// Signature getNetwork(): Promise<NetworkInfo> interface NetworkInfo { chainId: string; // e.g. "138" for Canton networkId: string; name: string; // e.g. "Canton Mainnet" } // Example const network = await jutis.getNetwork(); console.log(network.name); // → "Canton Mainnet"

getChainId

Returns the chain ID of the connected network.

const chainId = await signer.getChainId(); // → "138"

Events

Subscribe to provider events using on. All events follow the standard Ethereum provider event pattern.

interface JutisEvents { accountsChanged(accounts: string[]): void chainChanged(chainId: string): void disconnect(): void message(message: ProviderMessage): void }

accountsChanged

Fired when the user switches to a different connected account.

jutis.on("accountsChanged", (accounts) => { console.log("Switched to:", accounts[0]); });

chainChanged

Fired when the user switches the active network in the extension.

jutis.on("chainChanged", (chainId) => { console.log("Chain changed to:", chainId); // Reload dApp state if needed });

disconnect

Fired when the user explicitly disconnects your app from the provider.

jutis.on("disconnect", () => { console.log("User disconnected dApp"); // Clear local session state });

removeListener

Unsubscribe from an event.

const handler = (accounts) => { ... }; jutis.on("accountsChanged", handler); jutis.removeListener("accountsChanged", handler);

Error Codes

Jutis provider methods throw typed errors with a code field for programmatic handling.

CodeDescription
ACTION_REJECTEDUser explicitly rejected the request in the Jutis popup
CHAIN_NOT_FOUNDThe Canton network is not configured in the extension
NO_ACCOUNTSNo accounts connected — call requestAccounts first
UNKNOWN_ERRORUnexpected internal error — check message for details
try { await jutis.requestAccounts(); } catch (err) { if (err.code === "ACTION_REJECTED") { console.log("User said no"); } else if (err.code === "CHAIN_NOT_FOUND") { console.log("Please install Canton network in Jutis"); } }
