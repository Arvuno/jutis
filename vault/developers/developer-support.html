Integration Checklist

Before reporting an issue, verify each of the following in your integration:

  • check_circle window.jutis is detected before any API call
  • check_circle requestAccounts is called after a user gesture (button click)
  • check_circle All provider calls are wrapped in try/catch
  • check_circle Error code ACTION_REJECTED is handled as user cancellation
  • check_circle accountsChanged listener updates UI when account switches
  • check_circle chainChanged listener is subscribed after connection
  • check_circle Page reloads or re-initializes state after chainChanged
  • check_circle Jutis extension version is up to date (check chrome://extensions)

Bug Report Guide

Found a bug in the provider API? Send a complete report so we can reproduce and fix it.

1

Capture the error code and message

Log err.code, err.message, and err.stack from the caught exception.

2

Note your environment

Jutis extension version, browser (Chrome/Firefox/Edge), OS, and whether the issue reproduces on a fresh browser profile.

3

Provide a minimal reproduction

A stripped-down HTML file that triggers the bug with the fewest possible steps. Remove all dependencies.

4

Submit via the placeholder form

GitHub Issues link (placeholder — not yet active). Email: dev-support@jutis.io (placeholder).
