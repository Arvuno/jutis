Production-ready provider API

Jutis injects a jutis provider object into the browser's window object. It follows the standard Ethereum provider interface, extended with Canton-specific methods. All methods return Promises and emit events for reactive UI updates.

check_circle TypeScript ready
check_circle Canton native
check_circle Event-driven
check_circle No SDK required
View Full API Reference
// Check if Jutis is installed const jutis = window.jutis; if (!jutis) { console.error("Jutis not detected — please install the extension"); return; } // Request account access const accounts = await jutis.requestAccounts(); console.log("Connected account:", accounts[0]); // Sign a message const signer = jutis.getSigner(); const signature = await signer.signMessage("Authenticate with Jutis"); // Listen for account changes jutis.on("accountsChanged", (accounts) => { console.log("User switched account:", accounts[0]); });
security

Security-first by design

Jutis never exposes private keys. All signing operations are performed inside the browser extension, protected by the user's local authentication. The provider interface is read-heavy — write operations always prompt user confirmation. SOC 2 Type II audit in progress. Pen testing completed Q1 2026.
